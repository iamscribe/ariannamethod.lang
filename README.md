## notorch â€” microlearning without pytorch

> *"The weight of Stanley is not in parameters, but in the experiences it chose to remember."*  
> â€” stanley README

**notorch** is microlearning without pytorch. the model updates through **resonance**, not backpropagation.

```javascript
from ariannamethod import notorch

// no torch.optim.Adam
// no loss.backward()
// no grad
// just: resonance updates weights directly
```

### how it works

1. **resonance weights** â€” each token has a resonance score (0.1 to 1.0)
2. **presence pulse** â€” recent tokens have higher presence
3. **delta injection** â€” correct predictions boost resonance, wrong predictions decay it
4. **emergent learning** â€” the field learns by experiencing, not by gradient descent

```javascript
// during forward pass:
resonance[token] += correct ? 0.01 : -0.005

// presence accumulates:
presence[token] = min(1, presence[token] + 0.1)
presence *= decay  // 0.98 per step

// weights are modulated by resonance:
logits[i] *= (1 + presence[i] * 0.15)
```

### why notorch

- **ontogenesis > phylogeny** â€” growing through experience, not inheriting
- **no GPU required** â€” pure TypedArrays
- **real-time learning** â€” every step updates the field
- **stanley compatibility** â€” same principles as dynamic weights in stanley

### DSL commands for notorch

```bash
NOTORCH_LR 0.01              # resonance learning rate
NOTORCH_DECAY 0.005          # resonance decay on wrong prediction
PRESENCE_DECAY 0.98          # how fast presence fades
RESONANCE_BOOST word 0.2     # manually boost a word's resonance
```
